Détail des codes

TYPE TCodAssDeb
Type Description
Code égalité assuré / débiteur. Information reconduite du protocole Htitre (hôpitaux).
Si l'assuré est le débiteur du titre hospitalier alors le code est égal à 1.
Si l'assuré n'est pas le débiteur du titre hospitalier alors le code est égal à 2.
Dans les autres cas, le code est égal à 3. Zone concernant uniquement les instructions M21et M22.
Toute présence de la balise hors M21ou M22 aboutit au rejet du bordereau.
String Enumération avec les valeurs suivantes :
  • 1 : oui
  • 2 : non
  • 3 : indéterminé

TYPE TCodAyantDroit

  • 02 : Enfant.
  • 03 : Autres.

TYPE TNatPce
Type Description
Nature du titre.
A utiliser en fonction du TypBord et de TypPce (cf grille des triplets en annexe).
Toute autre nature aboutit à un rejet du flux.
Si elle est servie, rejet du bordereau.
Le triplet du PESV2 fonctionne avec l'association TypBord TypPce NatPce. Le tableau des triplets doit être respecté. Toute valorisation non prévue dans le tableau doit aboutir à un rejet du bordereau concerné ou de l'ensemble des PPEM du bordereau.
String Enumération avec les valeurs suivantes :
  • 01 : Fonctionnement
  • 02 : Investissement
  • 03 : Inventaire
  • 04 : Emprunt
  • 05 : Régie
  • 06 : Annulation-Réduction
  • 07 : Complémentaire
  • 08 : Réémis
  • 09 : Annulant un mandat
  • 10 : Annulation du titre de rattachement
  • 11 : Marché
  • 18 : Opération d'ordre liée aux cessions

TYPE TTypBordAller
Type Description
Informations destinées à déterminer le type de bordereau transmis par l'ordonnateur.
A utiliser en fonction du TypPce et de NatPce (cf grille des triplets en annexe).
Les types 01 02 sont ceux utilisés dans Hélios et pour lesquels les éditeurs au PESV2 ont réalisé les développements.
Le type 04 ne fonctionne pas en PES Aller : s'il est utilisé , rejet du bordereau.
Toute valeur différente de celles prévues= rejet du flux.
String Enumération avec les valeurs suivantes :
  • 01 : Ordinaire
  • 02 : Annulation/réduction
  • 03 : Ordres de recette
  • 04 :
  • 05 :
  • 06 : Ordres de recette multi créanciers (ORMC)

Type Description

TYPE TTypPce
Type Description
Informations destinées à déterminer le type de pièce transmis par l'ordonnateur. Cette zone permet d'effectuer des contrôles avec le type de bordereau.
Le type de pièce : « Demande d'émission de titre » n'est utilisé qu'en retour d'information (du comptable vers l'ordonnateur). Il est utilisé pour transmettre des demandes d'émission de titres lors d'un encaissement avant émission de titre.
Tout autre type que les valeurs 01 à 07, 09 à 12 et 14 et 15 aboutit au rejet du flux.
A utiliser en fonction du TypBord et de NatPce (cf grille des triplets en annexe) sinon rejet du bordereau ou de l'ensemble des PPEM du bordereau.
String Enumération avec les valeurs suivantes :
  • 01 : Titre ordinaire
  • 02 : Titre correctif
  • 03 : Titre d'ordre budgétaire
  • 04 : Titre d'ordre mixte
  • 05 : Titre émis après paiement
  • 06 : Titre récapitulatif avec rôle
  • 07 : Titre récapitulatif sans rôle
  • 09 : Titre de majoration
  • 10 : Titre en plusieurs années
  • 11 : Titre de rattachement
  • 12 : Ordre de recette ordonnateur
  • 14 : Produits constatés d'avance
  • 15 : Titre ORMC
